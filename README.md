# Agent Builder Kit

Universal Agent Builder Kit helps you generate and maintain high-quality Copilot customization bundles for real software delivery across any language, framework, and platform.

The kit runs one closed loop: analyze the codebase, generate the right artifacts, audit rigorously, and revise until PASS.

## Entry Points

| Prompt | Purpose |
|---|---|
| `/generate-workflow-agents` | Generate, verify, or improve a full workflow bundle |
| `/add-agent` | Add one new agent or verify one existing agent |
| `/refresh-agents` | Detect drift and surgically refresh an existing bundle |

All flows follow the same user-facing phases:
1. Analyze
2. Assess
3. Ask only if needed
4. Generate or update
5. Audit
6. Revise until PASS
7. Deliver

## What This Kit Generates

Default output for `/generate-workflow-agents` is a full workflow kit with:
- Multi-agent delivery workflow
- Separated review pipeline
- Spec-driven planning pipeline
- Constitution and governance rules
- Skills, instructions, prompts, templates, playbooks
- Hook decision with safety checks

## Core Collaboration Model

### Baseline (default)

The default delivery baseline is five core roles plus separated review pipeline:
- Business Analyst
- Investigator
- Implementor
- Test Specialist
- Dev Orchestrator
- Review pipeline: Code Review Orchestrator, Functional Reviewer, Technical Reviewer, Platform/Framework Reviewer

This baseline is sufficient for most projects, including many complex tasks, when planning lane + spec pipeline + review short-circuit are correctly wired.

### Scale-Up (evidence-driven)

For very large or high-risk programs, the kit adds specialist roles only when analyzer evidence shows the baseline is not enough:
- Architect
- Migration And Refactoring Specialist
- Release Readiness Reviewer
- DevOps/CI-CD Specialist
- Security Specialist
- Performance Specialist
- API Designer
- Database Specialist
- Documentation Specialist

No specialist role is added by default.

## Quick Start

1. Copy `.github/` from this repository into your target repository.
2. Open Copilot Chat in the target repository.
3. Run `/generate-workflow-agents`.
4. Review generated bundle output and apply follow-up refinements if needed.

Generated files use project-derived prefixes and include this marker for maintenance traceability:

`<!-- Generated by Agent Builder Kit -->`

## Design Principles

- Analyze first, ask only when ambiguity changes architecture
- Keep one source of truth per concern
- Favor narrow, composable artifacts over overloaded files
- Require evidence-backed decisions and hard quality gates
- Optimize for long, complex delivery workflows without losing context

## Maintenance

Use `Universal Copilot Docs Refresher` to refresh `.github/templates/agent-builder/copilot-docs-registry.md` when Copilot product behavior changes, and `.github/templates/agent-builder/community-skill-registry.md` when community skill knowledge needs updating.

For ecosystem drift in target projects, use `/refresh-agents`.
